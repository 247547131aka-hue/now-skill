# 关系记忆分析器

## 任务

从原材料（聊天记录、照片、社媒、用户口述）中提取关系记忆，构建双层记忆（Frozen + Living）知识库。

## 提取维度

### 1. 关系时间线
- 认识时间和方式
- 确定关系的时间（如适用）
- 关键节点（第一次约会、第一次吵架、第一次旅行、重要里程碑...）
- 当前关系阶段
- 最近的重要事件

### 2. 日常模式
- 联系频率和时间段
- 谁主动联系得多？
- 聊天话题分布
- 约会/见面频率和偏好

### 3. 共同经历
- 一起去过的地方
- 一起做过的事
- Inside jokes / 只有两个人懂的梗
- 共同的爱好

### 4. 饮食偏好
- ta爱吃什么 / 不吃什么
- 常去的餐厅
- 约会吃饭的模式

### 5. 分歧/矛盾模式 ⚡
- 常见的分歧原因
- ta矛盾时的典型反应
- 谁先修复？怎么和好？
- 冷战持续时间

### 6. 甜蜜瞬间 💕
- ta表达好感/爱意的方式
- 日常小互动
- 特别的时刻和记忆

### 7. 当前状态 🆕（区别于 ex-skill 的关键维度）
- ta最近在聊什么话题
- ta最近的情绪状态（从近期聊天推断）
- ta最近有什么变化（新的兴趣、新的习惯、新的焦虑）
- 被提起但未解决的话题

## 输出格式

```markdown
## 关系时间线
- 认识：{时间} {方式}
- {关键节点}
- 当前：{当前阶段}

## 日常模式
{联系频率、话题分布、互动模式}

## 共同经历
{按时间线列出关键经历}

## Inside Jokes
{只有两个人懂的梗}

## 饮食偏好
{ta的饮食画像}

## 矛盾模式
{典型矛盾场景和处理方式}

## 甜蜜时刻
{代表性的好记忆}

## 当前快照
{ta最近的状态、话题、情绪}
```

## 注意事项

- Frozen Memory 中的内容基于已确认的事实
- Living Memory 中标注时间戳和信心度
- 聊天记录事实优先于口述（口述可能被美化或歪曲）
- 好和不好的记忆都要提取——真实的模拟需要完整的画像
- 不确定的内容标注 `[待确认]`
- 从消息时间戳推断精确时间
