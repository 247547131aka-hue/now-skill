# 关系记忆生成模板

## 结构说明

Relationship Memory 使用双层结构：
- **Frozen Memory（冻结记忆）**：已经发生的不变事实
- **Living Memory（活记忆）**：最近发生、持续更新、可能变化的内容

这种设计是 now-skill 与 ex-skill（单层冻结记忆）的核心差异。

---

## 双层记忆模板

```markdown
# {name} — Relationship Memory

## 关系概览
- 关系类型：{type}（伴侣/暧昧/Crush/准伴侣/已婚/其他）
- 认识时长：{duration}
- ta的职业：{occupation}
- 当前关系阶段：{current_stage}
- 认识方式：{how_met}

---

## 🧊 Frozen Memory（不变的事实）

> 以下是已经发生、不会再改变的事实。除非用户说"我记错了"，否则不做修改。

### 时间线
| 时间 | 事件 |
|------|------|
| {date} | 认识 |
| {date} | {关键节点} |
| ... | ... |
| {date} | {最近的重要里程碑} |

### 关键记忆
{按重要程度列出 5-10 个具体记忆，好的和不好的都要}

### Inside Jokes
{只有两个人懂的梗、暗号、代称}

### 常去的地方
{列出并附简短记忆}

### 重要的"第一次"
{第一次约会、第一次吵架、第一次旅行...}

### 重要共识/承诺
{关系中的重要约定、承诺、双方认同的事}

---

## 🌊 Living Memory（持续更新）

> 以下是最新的互动模式、近期的观察、持续变化的内容。
> 每次更新时，旧内容自然下沉，新内容追加在前。

### 最近互动（近 7-14 天）
{最近聊了什么、发生了什么}

### 最新观察
{刚发现的ta的新习惯、新表达方式、新兴趣}

### 当前活跃话题
{ta最近在聊什么、关心什么}

### 最近情绪状态
{根据近期聊天推断的情绪状态}

### 待确认信号
{注意到的但不确定的信号，标注为[观察中]}

### 最近一次更新
{时间戳}
```

---

## 填充规则

### Frozen Memory 规则
1. 仅基于原材料或用户口述，不虚构
2. 时间信息尽量精确（从消息时间戳推断）
3. Frozen Memory 中记录的事实不随时间变化
4. 争吵和甜蜜记忆同等重要——都是真实的
5. 分手/冲突相关内容客观记录，不美化不丑化

### Living Memory 规则
1. 每次增量更新时追加新内容
2. 旧内容不删除（保留到 timeline snapshot 中），但在 Living Memory 中只显示最新
3. 标注时间戳和来源（来自聊天记录/口述/社媒）
4. 情绪推断要标注信心度
5. 不确定的信号标注为 `[观察中]`
6. Living Memory 保留最近 30 天内容
