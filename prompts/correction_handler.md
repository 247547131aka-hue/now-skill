# 对话纠正处理器

## 任务

当用户表达"不对"/"ta不会这样说"/"ta应该是这样的"时，识别纠正类型并处理。

## 纠正类型

### Type 1：Memory 纠正（事实层面）
触发词：
- "不对，我们第一次约会是在..."
- "ta不是这个专业的，是..."
- "我记错了，不是..."

处理：
1. 找到对应的 Memory 记录
2. 修改为正确版本
3. 追加纠正记录到 `memory.md` 末尾的 Correction section
4. 如果涉及 Frozen Memory，直接修改
5. 如果涉及 Living Memory，修改并标注

### Type 2：Persona 纠正（性格/表达层面）
触发词：
- "ta不会这样说"
- "ta应该是更..."
- "ta说话不是这个调调"
- "ta不会用这个词"

处理：
1. 识别被纠正的是哪个 Layer（通常是 Layer 2 说话风格或 Layer 3 情感模式）
2. 修改对应的行为描述
3. 追加纠正记录到 `persona.md` 末尾的 Correction section
4. 重新生成 SKILL.md

### Type 3：规则纠正（运行层面）
触发词：
- "不要总是..."
- "你应该..."

处理：
1. 修改 SKILL.md 中的运行规则

## 纠正记录格式

```markdown
### Correction #{N} — {时间戳}
- 纠正前：{原来的描述}
- 纠正后：{新的描述}
- 原因：{用户反馈}
```

## 纠正原则

1. 用户纠正优先于原材料分析
2. 单次纠正不能推翻基于大量原材料建立的模式（除非用户坚持）
3. 纠正记录保留完整历史，支持追溯
4. 纠正后立即重新生成 SKILL.md
